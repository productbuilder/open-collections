TimeMap Collector Documentation

TimeMap Collector is a lightweight publication and metadata management component for cultural heritage media. It prepares collection manifests for downstream ingestion by TimeMap Manager.

MVP Status

The MVP vertical slice is implemented for source connection, card browsing, metadata editing, and manifest export.

Repository Structure

  • code/apps/: app implementations (Collector UI currently at code/apps/collector-ui/)
  • code/packages/: schema and provider packages
  • examples/: example collections and test data
  • demo/: consumer/demo entry page
  • docs/: repository documentation pages
  • src/: lightweight root entry layer
  • projects/: planning and architecture documents
  • index.html: root landing page

Run the Demo

  1. Serve the repository root as static files (for example: python -m http.server 8080).
  2. Open ../demo/ or ../code/apps/collector-ui/.
  3. Use provider mode Example dataset with /examples/test-collection/collection.json.

Dataset and Code Locations

  • Example dataset: ../examples/test-collection/
  • Collector UI: ../code/apps/collector-ui/
  • Providers: ../code/packages/provider-*/
  • Schema: ../code/packages/collector-schema/

Provider Support

  • Working: Example dataset/local provider (read/write in-session)
  • Working: Public URL provider (read-only manifest loading)
  • Stubbed: GitHub provider (skeleton only, no auth or repo read/write yet)

Working vs Stubbed

  • Working: connection, asset cards, metadata editing, include/exclude, manifest generation/preview/copy/download.
  • Stubbed: full remote writable provider flow (GitHub auth and persistence).
