## Phase 2 requirements

### 1. Source badges on cards

Each asset card in the main collection viewport should clearly indicate which source it came from.

Requirements:
- each card shows a small source badge or label
- badge should be compact and readable
- badge should reflect the source label or provider in a useful way
- if both source label and provider are available, prefer the human-readable source label and optionally include provider subtly

Examples:
- `timemap-data`
- `GitHub`
- `Public URL`
- `Harbor archive`

The badge should not overwhelm the card.
Keep it lightweight but visible.

If a card already has other small metadata indicators, integrate the source badge cleanly with those.

---

### 2. Source filtering in the main workspace

Add a way to filter the merged asset grid by source.

Requirements:
- users can view assets from:
  - all sources
  - one specific source
- filtering should work without breaking selection/editing behavior
- source filter should use the current workspace sources list
- filter state should be reflected clearly in the UI

A practical implementation is acceptable:
- dropdown
- segmented control
- filter chips
- compact toolbar

Choose the simplest clean solution that fits the current app.

Recommended behavior:
- default = `All sources`
- selecting one source filters the visible grid
- selected asset handling should remain stable
- if the selected asset is filtered out, clear selection or handle gracefully

Do not overcomplicate with advanced search yet.

---

### 3. Improve the Sources manager dialog

The current Sources dialog should become more useful as a source manager.

Requirements:
- clearly list connected sources
- each source should show:
  - label
  - provider
  - status
  - capability badges if available (for example `Read`, `Read + Write`)
  - item count if available
- add clear actions per source, such as:
  - refresh
  - remove
  - inspect/configure (if practical)

The dialog should feel like a manager for multiple sources, not just a provider picker.

### Add source flow
Preserve or improve the add-source flow so users can still add:
- GitHub
- Public URL
- Example dataset

Planned providers may remain greyed out:
- Google Drive
- S3-compatible
- Wikimedia Commons
- Internet Archive
- others if already present in the UI

The main goal is that the list of connected sources becomes clear and useful.

---

### 4. Group metadata fields in the sidebar

The metadata editor currently contains fields, but it should become easier to read and work with.

Group the metadata form into sections such as:

#### Basic
- title
- description

#### Authorship
- creator
- attribution

#### Context
- date / period
- location

#### Rights
- license
- source

#### Classification
- tags

Requirements:
- keep current editable fields working
- improve readability and scanning
- use simple section headings and spacing
- do not overdesign
- preserve scroll behavior in the sidebar

Optional:
- make the top of the sidebar clearly identify the selected asset
- show source/provider info there if useful
