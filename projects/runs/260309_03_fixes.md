## Issues to fix

### 1. Source badge text on cards is too long
Right now cards show the full source URL or very long source strings.
This breaks or degrades the card layout.

This should be fixed so cards show a short, human-friendly source label instead of a full raw URL.

Examples:
- instead of `https://raw.githubusercontent.com/productbuilder/timemap-data/refs/heads/main/collection.json`
  show something like:
  - `timemap-data`
  - `GitHub`
  - `Public URL`
  - `Example dataset`

### 2. Source filter dropdown shifts and shows unnecessary extra text
Currently when selecting a source filter:
- the layout shifts left
- extra text like `Filtered: ...` appears
- the selected full source name is shown too verbosely

This should be simplified.

Desired behavior:
- filter control stays visually stable
- no layout jump when selecting a filter
- no redundant `Filtered: ...` label
- use short labels in the dropdown
- a simple dropdown is fine:
  - `All sources`
  - `timemap-data`
  - `Example dataset`

### 3. Source configuration is not remembered
When adding a Public URL source or GitHub source, the non-secret config is not being remembered.

We should remember safe source configuration locally so the app is easier to reuse.

We should NOT persist secrets like:
- GitHub PAT
- other tokens
- passwords

But we SHOULD persist non-secret source information such as:
- provider type
- label
- repo owner
- repo name
- branch
- folder path
- public manifest URL
- source mode
- read capabilities / metadata if useful

The app should be able to restore previously added sources when reopened/refreshed, or at least restore the source list/config in local browser storage.

## What to implement

### 1. Short source labels for cards

Introduce a source display label strategy.

Each source should have:
- a stable internal id
- a short display label for card UI
- optionally a longer detail label for source manager UI

Examples:
- GitHub repo source:
  - display label: `timemap-data`
  - detail label: `productbuilder/timemap-data`
- Public URL source:
  - display label derived from manifest filename, hostname, or user label
  - example: `timemap-data`, `archive.org`, `public collection`
- Example dataset:
  - `Example dataset`

Use the short display label on cards and compact UI areas.

Do not show full raw URLs on cards unless there is no better fallback.

If needed, add a helper function to derive source labels cleanly.

### 2. Simplify source filter UI

Refine the source filter control in the collection viewport.

Requirements:
- dropdown width should remain visually stable
- selecting a filter should not cause the surrounding layout to jump
- remove redundant text such as `Filtered: ...`
- show only the current compact filter selection
- keep asset counts if useful, but make them compact and non-disruptive

Example acceptable UI:
- dropdown: `All sources`
- small text next to it: `13 visible | 13 total`

Or:
- dropdown only, with counts elsewhere

The main requirement is: no long selected labels spilling into layout, and no extra “Filtered:” text line.

### 3. Persist safe source configuration locally

Add local persistence for source definitions/configuration, excluding secrets.

Use a browser-local persistence approach appropriate for the current app, for example:
- localStorage
- IndexedDB
- another lightweight local persistence mechanism already used in the project

Persist:
- source list
- source config except secret/token fields
- source display label
- provider type
- repo owner/repo name/branch/path
- public URL manifest
- source status if useful
- workspace source metadata needed to reconstruct the source list

Do NOT persist:
- GitHub PAT
- OAuth tokens
- passwords
- secret credentials

### 4. Restore remembered sources on load

On app startup, restore remembered source definitions into the Sources manager.

Desired behavior:
- previously added non-secret sources appear again
- app can show them in the Sources dialog
- user can reconnect/reload them
- if a source requires a secret/token, the UI should clearly indicate that credentials must be re-entered if needed

For example:
- remembered GitHub source shows:
  - provider = GitHub
  - owner/repo/branch/path restored
  - token field blank
  - user can re-enter token and reconnect
- remembered Public URL source can likely reconnect directly if no secret is needed

### 5. Preserve current working behavior

Do not break:
- card selection
- metadata sidebar
- asset viewer
- multi-source workspace
- source manager
- manifest dialog
- existing provider loading logic

This is a cleanup/improvement pass.

## UX guidance

### Cards
Use:
- short source badge
- no raw full URL

### Source manager
It is okay for the source manager to show more detail than cards.
For example:
- cards: `timemap-data`
- source manager: `GitHub · productbuilder/timemap-data @ main:/`

### Remembered sources
A good pattern is:
- show restored sources on load
- mark them as:
  - connected
  - needs reconnect
  - read-only public
depending on what can be restored safely

This is especially important for GitHub sources without stored PATs.
