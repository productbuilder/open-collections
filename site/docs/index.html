TimeMap Collector Documentation

The main UI is now a reusable Shadow DOM Web Component: timemap-collector.

1. Embedding

<script type="module" src="/code/apps/collector-ui/src/index.js"></script>
<timemap-collector></timemap-collector>
  • Custom element name: timemap-collector
  • Uses Shadow DOM for style and shell encapsulation
  • Main layout: white header, card viewport, metadata sidebar

2. SaaS Shell Layout

  • Header: title + actions for Sources and Manifest dialogs
  • Main workspace is height-bounded with split panes
  • Left pane: independently scrollable media card viewport
  • Right pane: independently scrollable metadata editor
  • Sources manager and Manifest UIs in dialogs

3. Source architecture in MVP

  • Sources manager with connected-source list and add-source forms
  • Multiple read sources can be loaded into one workspace
  • GitHub source (manifest-first import; inline item metadata save with PAT write permission)
  • Public URL source flow (read-only)
  • Example/local source flow
  • Disabled placeholders for Google Drive, S3, Wikimedia Commons, Internet Archive
  • Merged asset browsing and source-aware selection
  • Source badges on cards and source filtering in the main workspace
  • Asset viewer dialog for large preview (View button or double-click)
  • Metadata editor grouped by Basic, Authorship, Context, Rights, Classification
  • Remembered source definitions in local browser storage (safe config only; no tokens)
  • Manifest generate/preview/copy/download
