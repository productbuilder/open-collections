Tools

The protocol defines how collections are published on the web. These tools help you create, publish, discover, and present those collections in day-to-day work.

For non-technical teams, this means practical workflows. For technical teams, the same tools can be embedded and integrated into existing systems.

Collection Manager

Collection Manager helps people assemble and edit collection metadata, connect storage providers, and publish collection outputs.

  • Use it as a standalone tool for collection publishing workflows.
  • Embed it as a web component inside websites, portals, apps, or collection systems.
  • Supports integration through a flexible attribute/configuration API for embedding contexts.

Collection Browser

Collection Browser provides a read-focused interface for exploring published collections and item details.

  • Use it as a standalone viewer for public or internal audiences.
  • Embed it as a web component where you want collection browsing in an existing UI.
  • Supports a flexible attribute/configuration API for embedded experiences.

Indexer

Indexers crawl published manifests, item URLs, and media references so collections can be searched, aggregated, and discovered across domains.

Registrator

Registrator services help organizations announce and curate published collection endpoints, improving visibility and trust across networks.

TimeMap

TimeMap turns published collection data into timeline and map exploration views, helping users understand historical and geographic context.

How tools relate to the protocol

Open Collections Protocol is the shared publishing contract. Tools are optional, interoperable implementations that work with that contract.

You can adopt one tool, several tools, or build your own tooling while still remaining compatible with the same open web surface.
